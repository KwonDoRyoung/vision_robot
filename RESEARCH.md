

# 목표
- 카메라(RGB / Depth) 와 라이더(LiDAR)를 이용한 실시간 자율주행 로봇 구현
- 세부 목표
    - 카메라 다중 객체 탐지 (여기서 다중객체는 동적 객체로써 사람으로 한정함. 따라서 나머지 객체는 무시)
    - 카메라에서 다중 객체 탐지 후, LiDAR 를 통한 정확한 위치 및 속도(방향 포함) 파악
    - SLAM 주행 및 Policy에 따라 주행 진행(회피(좌/우 회전), 좌로 이동, 우로 이동, 정지, 정상 주행, 재자리 회전 등)
    - 주행 영상 녹화 (rosbag 형태) * 추후 진행

# 구현해야될 것 
- RGB 이미지 기반 객체(사람) 감지 노드를 통해 bounding box 정보를 전달
- depth 를 통해 대략적인 위치 파악 (필요할까?)
- 객체 탐지 결과와 라이다 값을 토대로 각 객체의 속도 위치 정보를 발행
- 객체들의 속도와 위치 정보를 토대로 로봇 주행을 위한 정책 단에 전달
- 정책 단에서 로봇의 구동부 제어(이 부분은 이번에 작성은 하지 않는다.)
1. Yolo26n TensorRT 기반 객체 탐지 ([Ultralitic](https://docs.ultralytics.com/guides/nvidia-jetson/))
    - 객체를 놓칠 수도 있기 때문에 N 프레임(설정) 정도는 이전 결과를 활용하는게 나을 듯함.
2. 탐지한 객체의 LiDAR 포인트를 구하여 위치 및 속도 추출
    - LiDAR 는 3D 포인트이며, Yolo 객체 탐지 bounding box 는 2D 이다. 
      (공간 상에 구에 z 축과 평행한 면에 yolo bounding box 가 그려진다고 생각된다.)
      이에 대한 의견 필요
    - 너무 많은 포인트를 확인하면 처리 속도가 느려질 것이다. 객체 탐지 박스안에 잡힌 포인트 중 S 개만 샘플링하여 진행하는게 나을 듯 하다.
      너무 적으면 오차가 생길거고 너무 많으면 지연이 발생하여 주행에 어려움을 겪을 것으로 판단됨.
3. 로봇이 주행 중에 해당 정보를 수집하고 이에 대한 판단을 policy 로부터 한다.
    - 정면에서 객체가 다가온다면, 정지(가능하다면, 측면으로 이동)
    - 정면 중에서도 로봇의 정면이 아니라면, 해당 객체와 거리를 띄우는 방향으로 이동(자연스러운 회전)
    - 다가오는 객체는 로봇에 위험 또는 다가오는 객체에 위험이 되기 때문에 이것이 중요함.
